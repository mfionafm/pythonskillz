# Google Foobar Algorithmic Engineering Portfolio

A collection of hyper-optimized, production-ready Python services designed to solve high-impact structural bottlenecks, data transformations, and complex combinatorial scale challenges.

## 📌 Strategic Overview
The projects contained within this repository represent successful solutions to the legendary, invite-only **Google Foobar Challenge**. Each module addresses a unique algorithmic hurdle where naive brute-force implementations fail due to exponential memory consumption, time-limit constraints, or system crashes. 

Instead of relying on over-engineered libraries or fragile scripts, these solutions are built from **First Principles**—focusing on system invariants, structural modeling, and defensive architecture to achieve deterministic system closure.

---

## 📂 Repository Architecture

This portfolio is systematically organized into five distinct service layers, moving from clean string parsing up to dense group theory modeling:

### [01. Grade 1 Braille Translator](./01_braille_translator/)
* **The Challenge:** Linear text transformation requiring specific case-modification and numerical state-shifting protocols.
* **The Solution:** A performant microservice that implements strict defensive error handling, safely bypassing unknown characters (like emojis) without halting system execution.
* **Core Script:** `translator.py`

### [02. Volatile Mutation Optimizer](./02_bomb_baby_optimizer/)
* **The Challenge:** Verifying generation boundaries for massive target matrix coordinates (up to $10^{50}$) without getting trapped in a multi-trillion-path decision tree.
* **The Solution:** An optimization script utilizing **directional inversion** (working backward from target to origin) and **asymptotic fast-forwarding** via floor division and modulo operators.
* **Core Script:** `optimizer.py`

### [03. Network Flow & Bottleneck Matrix Router](./03_escape_pods_network/)
* **The Challenge:** Computing maximum network throughput across a highly redundant, multi-ingress and multi-egress directed graph network prone to local saturation.
* **The Solution:** A maximum-flow router modeled on the Ford-Fulkerson algorithm that normalizes complex matrices via a virtual "Super-Source" (Basement) and "Super-Sink" (Roof) while enforcing a strict integer infinity compilation barrier.
* **Core Script:** `solution.py`

### [04. Combinatorial Equivalence Enumerator](./04_disorderly_escape_symmetry/)
* **The Challenge:** Evaluating a massive raw search space ($20^{144}$ combinations) for unique grid states where spatial permutations (rotations, flips, row/col swaps) render configurations identical.
* **The Solution:** A compressed representation framework using integer partitions to map permutations into conjugacy classes, combined with a local GCD-based orbit interaction kernel.
* **Core Script:** `enumerator.py`

### [05. Product Group Action Evaluator](./05_grid_symmetry_evaluator/)
* **The Challenge:** The mathematically rigorous, peer-reviewed culmination of the grid equivalence problem.
* **The Solution:** A fully decoupled, canonical execution of Burnside's Lemma over the direct product of symmetric groups ($S_w \times S_h$). It isolates the mathematical logic into dedicated, testable functions with absolute O(1) factorial lookup performance.
* **Core Script:** `evaluator.py`

## ⚙️ Core Architectural Pillars
Across all five codebases, the system architecture adheres to four strict engineering constraints:
1. **Defensive Safety Nets:** Systems are explicitly bounded at the entry point to instantly flag and reject mathematically invalid data before it hits the core engine.
2. **Encapsulation & Scope Control:** Code utilizes secure variable scoping, argument injection, and closures to prevent global dependency leakage.
3. **Performance Awareness:** Volatile loops are systematically extracted and replaced with high-speed mathematical invariants (like GCD loops and floor division) to ensure lightning-fast execution.
4. **Cognitive Scaffolding:** Code documentations preserve explicit trade-off analyses and mental anchors, bridging intuitive mechanical concepts directly to standard academic theorems.
