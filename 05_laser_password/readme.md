# Beatty Sequence Reduction Engine
An asymptotically efficient arithmetic system that computes exact, non-overlapping sequence sums across arbitrary-precision domains up to 101-digit integers.

## 📌 Project Overview
This project delivers a mathematically rigorous, structurally compressed solution to an exponential calculation bottleneck. Standard iterative traversal algorithms (\(\mathcal{O}(N)\)) collapse when evaluating massive inputs due to catastrophic processing time constraints. 

Evaluating the summation:
\[\sum_{i=1}^{n} \lfloor i\sqrt{2} \rfloor\]

for inputs approaching \(n = 10^{100}\) is computationally intractable under ordinary linear scans. This service resolves the scaling bottleneck by exploiting the recursive symmetry of complementary Beatty sequences, transforming an effectively impossible traversal into a rapidly collapsing logarithmic mirroring process that operates entirely through integer arithmetic.

## Technical Specifications
* **Input Constraints:** String representation of integer \(n\) up to 101 digits long.
* **Theoretical Framework:** Formal application of Rayleigh's (Beatty's) Theorem over complementary irrational scale dimensions.
* **Performance Profile:** Executes in \(\mathcal{O}(\log n)\) time. Each recursive depth iteration compresses the active problem space by approximately \(\sqrt{2} - 1 \approx 0.4142\), reducing runtime to negligible practical scales for typical inputs.

## ⚙️ Core Logic Flow
The engine enforces absolute mathematical precision across an arbitrary-precision integer pipeline:

### 1. Complementary Beatty Mapping
The system leverages the complementary structure between irrational Beatty sequences generated by \(\sqrt{2}\) and its unique interlocking companion space \(2 + \sqrt{2}\) under the balancing identity:
\[\frac{1}{p} + \frac{1}{q} = 1\]
which induces a complete, non-overlapping partition of the positive integers via Beatty complementarity, allowing the original summation space to be recursively mapped into a smaller companion domain.

### 2. Recursive Domain Collapse
Each recursive pass executes a modular step transformation, computing:
\[n' = \lfloor (\sqrt{2} - 1)n \rfloor\]
which slashes the active problem space by more than half every iteration. The engine implements the recursive summation identity:
\[S(n) = nn' + \frac{n(n+1)}{2} - \frac{n'(n'+1)}{2} - S(n')\]
This converts an impossible-scale traversal problem into a compact, cascading reduction chain.

### 3. Fixed-Point Precision Kernel
To bypass standard floating-point precision loss limits and eliminate instability, the implementation operates entirely through scaled integer arithmetic using a 105-digit high-precision integer expansion of \(\sqrt{2} - 1\). The recursive transform step:
```python
n_prime = (n * SQRT2_MINUS_1_O100) // (10**101)
```
preserves exact floor behavior while maintaining constant-time execution per recursion step under fixed precision scaling.
